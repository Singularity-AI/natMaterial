Background

  1. Sulla problematica: Interconversion between #FHIR and HL7 v2 (by Grahame Grieve). Broadly there are 2 approaches to this conversion:
    • The creation of a FHIR message bundle that mirrors the contents of the v2 message, and is intended to be an equivalent representation, behaving in the same way as v2 messages.
    • Using the contents of the v2 message to update a FHIR server – perhaps extracting Encounter resources or creating a Bundle that is intended to act as a ‘transaction’ bundle against a FHIR server. I think this will be a much more common use case.

  2. Getting Sample HL7 Data
  3. Informazioni specifiche sul mapping FHIR/HL7v2 si trovano nella documentazione FHIR delle risorse.
    Si veda ad esempio: Patient: 8.1.14.4 Mappings for HL7 v2 Mapping (http://hl7.org/v2) .

  4. Un inquadramento di come usare queste informazioni:
    • From the FHIR Patient documentation page, click on Mappings, and search for HL7 v2 (this leads to patient-mappings.html#v2)
    • You'll see that most of the FHIR resource properties have corresponding HL7 v2 segment and field mappings. For example, the Patient.identifier property is mapped to the PID-3 field. But how exactly does each individual PID-3 component map into the identifier property?
    • Return to the Patient Content tab, and look at the structure of the resource. You'll see that the "identifier" property is of type Identifier.
    • Click on Identifier to navigate to the documentation page for that data type: datatypes.html#Identifier.
    • Then click on Mappings to view the HL7 v2.x mappings for that data type: datatypes-mappings.html#Identifier.
      This should give you enough information to be able to map pieces of the HL7 message into the corresponding FHIR resource properties.

      Fir an example using MIRTH, see MIRTH HL7 v2.x to FHIR Conversions (R4)

      Una discussione sul passaggio (EHR-)HL7v2-FHIR usando Mirth si trova anche in: HL7 v2 to FHIR Interface Mapping
  5. Molto utile sembra anche v2-to-FHIR Map Inventory che conduce a tabelle quali
    • HL7 Message - FHIR R4: ADT_A01
    • TAB SEGMENT (da cui poi, ad esempio HL7 Segment - FHIR R4: NK1[Patient])
    • TAB DATA TYPE
    • TAB Code system
